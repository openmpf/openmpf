Edit MPF Properties

This view allows an admin user to modify MPF properties. MPF properties consist of two types. The General System Properties are configured at startup, and would require a restart of OpenMPF to affect a change to those properties. The Default Job Properties are configured at startup, but may be changed by an administrator. If the change is applied the updated value will be used going forward for all current and future jobs. There are buttons to reset and save the General System Properties in the bottom table, and buttons to reset, apply and save the Default Job Properties in the top table. It is recommended that General System Properties only be changed and saved when there are no jobs being processed. An administrator will need to restart the web application for modified General System Properties to take effect. Restart is not required to apply Default Job Properties. This view shows the values of MPF properties.

In the table below, the properties can take on several states:

  • Properties in black are values currently running on MPF.
  • Any property in orange marked with have been modified but have not been saved to the server. Each "Reset" button and the "Reset All Properties" button will only reset these properties to the last saved value on the server.
  • General System Properties in blue and/or marked with have been saved to the server, but will only take effect after the server is restarted.
